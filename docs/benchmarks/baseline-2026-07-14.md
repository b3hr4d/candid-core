# Initial benchmark baseline — 2026-07-14

This is the first measurement from the Issue #36 benchmark implementation. It is a local observation and reproducibility example, not a portable performance guarantee or CI threshold.

## Environment

- Host: Apple `Mac16,11`, 12 logical CPUs, 24 GiB RAM
- OS: macOS 26.5.2 (25F84)
- Rust: `rustc 1.91.1 (ed61e7d7e 2025-11-07)`, `aarch64-apple-darwin`
- Cargo: 1.91.1
- Repository base before the Issue #36 working tree: `b53de1edd84c464d5b06a61eab71b8635010f6f2`
- Criterion: 0.5.1, 20 samples, 1 second warm-up, 3 second measurement
- Command: `cargo bench --bench compilation --locked -- --noplot`

The tables use Criterion's central estimate. Several 20-sample groups reported outliers, so these values should be compared only after repeating the run under similar host conditions.

The recursive-chain result is the clearest initial investigation target for Issue #6. This benchmark does not diagnose the cause or authorize an algorithm change; it only makes the scaling cost visible.
